# 기술 보고서: NASDAQ 하방위험 오버레이 v4

**작성자:** Quant Analysis Team  
**버전:** v4  
**용도:** 포트폴리오 리스크 오버레이 / 면접 포트폴리오 리뷰

## 1. 연구 목표
이 프로젝트의 목표는 시장 상승을 무조건 최대한 따라가는 것이 아닙니다.  
핵심은 하방위험이 높아지는 시점을 식별하고, 낙폭이 깊어지기 전에 NASDAQ 노출을 줄이는 것입니다.

이 프레이밍은 다음 세 가지를 바꿉니다.
- 타깃 정의
- 검증 기준
- 성과 해석 방식

## 2. v4 가설
현재 작업 가설은 다음과 같습니다.

1. 유동성, 금리, 변동성, 크레딧 스트레스에는 단기 하방위험 정보가 포함되어 있다
2. 원시 클래스 라벨보다 calibration된 위험확률이 더 유용하다
3. 노출 제어는 국면 시그널과 변동성 스케일링을 함께 써야 한다
4. 최종 오버레이는 buy-and-hold뿐 아니라 평균 노출이 같은 단순 de-risking 벤치마크도 이겨야 한다

## 3. 데이터와 피처
모델은 다음 데이터를 사용합니다.
- NASDAQ
- VIX와 MOVE
- FRED 유동성 및 대차대조표 데이터
- 금리, HY 스프레드, 금융 스트레스 지표
- 유동성 베타 분석용 대형주 일부

v4의 피처 처리 방식은 다음과 같습니다.
- 유동성 변수는 주로 수익률 형태를 유지
- 거시 스트레스 변수는 rolling z-score로 변환
- 시차가 있는 lagged feature를 사용해 시점 규율을 유지

이 목적은 원시 금리/스트레스 레벨값을 트리 모델에 바로 넣을 때 생기는 레짐 붕괴 문제를 완화하는 데 있습니다.

## 4. 타깃 설계
라벨은 20거래일 horizon, -5% threshold 기준의 하방위험 이벤트입니다.

v4는 단순히 현재 종가 대비 미래 최저점만 보는 방식이 아니라, **true forward drawdown** 로직을 사용합니다.  
이 점이 중요한 이유는, 모델이 단순한 종점 약세가 아니라 경로 의존적인 drawdown 위험에 반응해야 하기 때문입니다.

## 5. 검증 구조
### 5.1 누수 통제
- purged 시계열 교차검증
- embargo window
- 파이프라인 내부 scaler
- 익일 집행 기준

### 5.2 모델 학습
- 주력 분류기로 XGBoost 사용
- 베이스라인으로 로지스틱 회귀 사용
- `scale_pos_weight` 기반 클래스 불균형 대응
- inner CV도 일반 `TimeSeriesSplit`이 아니라 purged 구조 유지

### 5.3 확률 처리
- 먼저 out-of-fold 확률을 생성
- calibration 계층 적용
- EMA smoothing은 fold 내부에서만 수행
- hysteresis와 exposure mapping은 calibration된 확률 위에 얹음

이 구조는 모델 점수와 실제 매매 의사결정 계층을 분리해 줍니다.

## 6. 트레이딩 계층
최종 트레이딩 프로세스는 다음과 같습니다.

1. 하방위험 확률 추정
2. 확률 경로 smoothing
3. 확률을 시그널 / 노출로 변환
4. 실현 변동성 기준으로 노출 스케일링
5. 익일 집행
6. turnover 반영 거래비용 차감

v4의 비용 모델은 단순 시그널 전환 횟수가 아니라 **목표 노출 변화량(turnover)** 을 기준으로 합니다.

## 9. 민감도와 진단
### 9.1 타깃 민감도
현재 그리드에서는 `20D / -5%` 사양이 가장 강한 타깃으로 유지됩니다.  
더 긴 `40D` 타깃은 PR-AUC 수치 자체는 개선되지만 포트폴리오 결과는 약화됩니다.

### 9.2 시그널 민감도
연속 노출 매핑을 넣은 뒤에는 signal sensitivity가 더 이상 완전히 평평하지 않습니다.  
다만 변화 폭은 아직 제한적이며, 이는 모델이 어느 정도 강건하다는 뜻이면서 동시에 확률 범위가 여전히 압축돼 있다는 뜻이기도 합니다.

### 9.3 피처 그룹 제거 실험
feature-group ablation은 최종 PnL보다 PR-AUC를 더 크게 바꿉니다.  
즉, 피처 블록별로 ranking 품질 차이는 존재하지만 그 차이의 상당 부분이 exposure-conversion 계층에서 흡수되고 있습니다.

### 9.4 위험 Decile
`risk_decile_summary_v4.csv`를 보면 realized event rate 기준 확률 ranking이 완벽하게 단조적이지는 않습니다.  
이는 중요한 경고 신호입니다. calibration이 해석 가능성을 높였을 수는 있지만, 확률 표면은 아직 충분히 매끄럽지 않고 decile monotonicity도 불완전합니다.

## 10. 한계
1. 2023년 이후 레짐 집중은 여전히 실제 한계다
2. PR-AUC는 위험 이벤트 비중 대비 소폭 개선에 그치며, 확률 ranking의 질은 아직 modest하다
3. feature ablation과 signal sensitivity에는 여전히 일부 압축이 남아 있다
4. 최종 오버레이의 절대수익은 buy-and-hold보다 약간 낮다
5. liquidity-beta 분석은 아직 직접 실행 로직보다는 설명용 성격이 더 강하다
